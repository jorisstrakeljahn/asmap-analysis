# ASmap Data Analysis

Analysis of ASmap files generated by [Kartograf](https://github.com/asmap/kartograf),
compared against collaborative-run baselines from [asmap-data](https://github.com/bitcoin-core/asmap-data).

## What I did

Setup via Nix (`nix develop` in the Kartograf repo), then:

1. RPKI-only run (`./run map`) — 719,680 entries, 6:41 runtime
2. Full run with all sources (`./run map -irr -rv`) — 1,352,588 entries, 11:55 runtime
3. Three diffs against the March 2026 baseline (epoch 1772726400) from asmap-data

All runs on 2026-04-06, Kartograf v0.4.13, rpki-client 9.7.

## Results

| Comparison | Diff entries | What it isolates |
|---|---|---|
| RPKI-only vs baseline | 16,821 | source difference + time drift (mixed) |
| Full run vs baseline | 20,100 | time drift only (same sources, ~1 month apart) |
| Full run vs RPKI-only | 316,863 | source difference only (same day) |

Running a full run and an RPKI-only run on the same day lets you separate
two effects that are otherwise mixed: changes from using different data
sources vs. changes from the internet evolving over time.

## What surprised me

**RPKI and IRR/Routeviews almost never disagree.** Out of 316,863
differences between the full run and RPKI-only run, only 1 is an actual
conflict where both maps have the same prefix mapped to a different AS.
The other 316,862 are coverage gaps — prefixes that IRR/Routeviews cover
but RPKI doesn't.

This means the three sources complement each other rather than verify
each other. The value of IRR and Routeviews is not "catching RPKI errors"
but "covering the ~47% of prefixes that RPKI doesn't reach yet."

**RPKI data changes within minutes.** My two runs were 20 minutes apart.
The RPKI download hashes differ — 14 additional ROA files appeared. The
practical impact was tiny (1 changed entry), but for collaborative
launches that depend on hash equality, even small changes break consensus.

## Where tooling and documentation caused problems

**No progress feedback during RPKI download.** The download phase takes
~5 minutes with no output. `tqdm` is already a dependency and used during
RPKI validation (`kartograf/rpki/fetch.py` line 149), but not during
download (lines 88-95) because `rpki-client` runs as an external
subprocess with `capture_output=True`.

**The diff output format is undocumented.** `asmap-tool.py diff` produces
three line formats depending on whether a prefix is reassigned, added,
or removed. Understanding which is which requires reading the source
(`bitcoin/contrib/asmap/asmap-tool.py` lines 158-163). I wrote
[`scripts/analyze_diff.py`](scripts/analyze_diff.py) to parse and
summarize these diffs.

## What this means for the dashboard project

Three metrics that would be useful based on what I found:

- **RPKI adoption per AS.** The largest ASes in the coverage-gap diff
  (AS4134/China Telecom, AS174/Cogent, AS3549/Level3) haven't adopted
  RPKI yet. Tracking this over time shows adoption progress.
- **Diff size between consecutive maps.** How much does the map change
  per month? The baseline comparison shows ~20k reassignments in one
  month — is that typical or an outlier?
- **Source contribution ratio.** What fraction of the final map comes
  from each source? Currently ~53% of entries come from IRR/Routeviews.
  As RPKI adoption grows, this ratio should shift.
